## 🚀 Proyectos

**⚽Base de Datos Relacional — Sistema de Competencias Esports:**

[![repo](https://img.shields.io/badge/Base_Datos_SQL--Sistema_de_Competencias-181717?style=for-the-badge&logo=github&logoColor=white)](https://github.com/krysvill/Bootcamp_Data_Driven_Decisions_Specialist/tree/b3ec4d3b40d86078e0a0062d0ced85635f102468/Sistema%20de%20Competencias%20Esports)

- Diseñé e implementé una base de datos relacional en MySQL para gestionar competencias de esports, con 4 tablas normalizadas: Competencias, Equipos, Jugadores y Registros_Competencias.
- Definí integridad referencial mediante claves foráneas (FK) entre tablas, tipos de datos específicos por campo (DECIMAL, TINYINT, DATE) y restricciones NOT NULL para garantizar la calidad del dato desde la estructura.
- El modelo permite registrar resultados por equipo y competencia (victoria/empate/derrota), rendimiento numérico, posición local/visitante y duración real de cada partido, cubriendo las necesidades analíticas del negocio.

```Excel``` ```SQL``` ```Diseño Relacional```

**🚕 RideFare — Análisis de Tarifas:** 

[![repo](https://img.shields.io/badge/RideFare--Analisis_de_Tarifas-181717?style=for-the-badge&logo=github&logoColor=white)](https://github.com/krysvill/Bootcamp_Data_Driven_Decisions_Specialist/tree/b3ec4d3b40d86078e0a0062d0ced85635f102468/RideFare%20-%20Analisis%20de%20Tarifas%20en%20servicios%20de%20Movilidad)

- Análisis exploratorio (EDA) sobre +693.000 registros de viajes (Uber/Lyft) cruzados con datos meteorológicos mediante merge por zona geográfica y franja horaria (redondeo de epoch timestamp a hora).
- Se realizó limpieza avanzada: detección de nulos y duplicados, conversión de timestamps epoch (segundos y milisegundos), e imputación de precios faltantes segmentada por rangos de distancia usando medianas por intervalo.
- Generé visualizaciones ejecutivas (heatmap de correlaciones, boxplots, scatterplots, series de tiempo) identificando que la distancia es el principal predictor de precio, con variaciones por hora pico y clima de hasta un 10%.
- Proyecto Grupal presentado ante panel de mentores especializados al cierre del módulo de Python — ESPOL (2025-2026).

 ```Python``` ```Pandas``` ```NumPy``` ```Seaborn``` ```Matplotlib```  ```EDA ```
